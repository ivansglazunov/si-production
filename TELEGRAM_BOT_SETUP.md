# Инструкция по настройке Telegram бота si_production_bot

## 📋 Что нужно предоставить

Для работы бота вам необходимо предоставить следующую информацию:

### 1. Токен бота
+
1. Откройте Telegram и найдите бота [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям и создайте бота с именем `si_production_bot`
4. BotFather предоставит вам **токен бота** (выглядит как `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`)
5. Сохраните этот токен

### 2. ID группы si_production_reception

**Важно:** Группа должна быть:
- Супергруппой (Supergroup)
- С включенными топиками (Topics/Forum mode)

#### Как получить ID группы:

**Способ 1: Через бота @userinfobot**
1. Добавьте бота @userinfobot в вашу группу
2. Отправьте любое сообщение в группу
3. Бот ответит с информацией, включая ID группы (будет отрицательным числом, например: `-1001234567890`)

**Способ 2: Через готовый скрипт (рекомендуется)**
1. Убедитесь, что бот добавлен в группу
2. Запустите скрипт:
```bash
TELEGRAM_BOT_TOKEN="ваш_токен" node scripts/get-group-id.js
```
3. Отправьте любое сообщение в группу
4. Скрипт выведет ID группы, который нужно добавить в `.env`

**Способ 3: Через веб-версию Telegram**
1. Откройте группу в веб-версии Telegram (web.telegram.org)
2. URL будет содержать ID группы

### 3. Настройка группы

1. **Включите топики (Topics) в группе:**
   - Откройте настройки группы
   - Перейдите в "Topics" или "Топики"
   - Включите функцию топиков

2. **Добавьте бота в группу:**
   - Добавьте `@si_production_bot` в группу `si_production_reception`
   - Сделайте бота **администратором группы** (это необходимо для создания топиков)

3. **Проверьте права бота:**
   - Бот должен иметь права на:
     - Отправку сообщений
     - Создание топиков
     - Управление сообщениями (опционально)

## 🔧 Настройка проекта

### Шаг 1: Установка зависимостей

```bash
npm install
```

### Шаг 2: Настройка переменных окружения

1. Скопируйте файл `.env.example` в `.env`:
```bash
cp .env.example .env
```

2. Откройте `.env` и заполните значения:
```env
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
TELEGRAM_RECEPTION_GROUP_ID=-1001234567890
```

### Шаг 3: Настройка веб-хука (для production)

Для работы в production нужно настроить веб-хук:

1. **Убедитесь, что ваш сервер доступен из интернета** (используйте ngrok для разработки или настройте домен для production)

2. **Установите веб-хук:**
```bash
curl -X POST "https://api.telegram.org/bot<YOUR_BOT_TOKEN>/setWebhook" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://your-domain.com/api/telegram/webhook"}'
```

3. **Проверьте веб-хук:**
```bash
curl "https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getWebhookInfo"
```

### Шаг 4: Запуск бота

#### Для разработки (с ngrok):

1. Запустите Next.js сервер:
```bash
npm run dev
```

2. В другом терминале запустите ngrok:
```bash
ngrok http 3000
```

3. Используйте URL от ngrok для настройки веб-хука (см. Шаг 3)

#### Для production:

1. Соберите проект:
```bash
npm run build
```

2. Запустите сервер:
```bash
npm start
```

3. Настройте веб-хук на ваш production URL

## 🧪 Тестирование

1. Откройте Telegram и найдите вашего бота `@si_production_bot`
2. Отправьте команду `/start`
3. Проверьте, что в группе `si_production_reception` создался топик с вашим username
4. Отправьте боту любое сообщение
5. Проверьте, что сообщение появилось в соответствующем топике группы

## 📁 Структура файлов

- `lib/telegram-bot.js` - Основная логика бота
- `pages/api/telegram/webhook.js` - API роут для обработки веб-хуков
- `data/telegram-topics.json` - Кеш маппинга userId -> topicId (создается автоматически)

## 🔍 Устранение неполадок

### Бот не отвечает

1. Проверьте, что токен правильный
2. Убедитесь, что веб-хук настроен правильно
3. Проверьте логи сервера на наличие ошибок

### Ошибка "Chat not found"

1. Убедитесь, что бот добавлен в группу
2. Проверьте, что ID группы правильный (должен быть отрицательным числом)
3. Убедитесь, что группа является супергруппой

### Ошибка "Topics are not enabled"

1. Включите топики в настройках группы
2. Убедитесь, что группа является форумом (супергруппой с топиками)

### Ошибка "Not enough rights"

1. Сделайте бота администратором группы
2. Убедитесь, что у бота есть права на создание топиков

## 📝 Примечания

- Бот автоматически создает топик для каждого пользователя при первом сообщении
- Название топика формируется на основе username пользователя (если есть) или его ID
- Маппинг userId -> topicId сохраняется в файле `data/telegram-topics.json`
- Все сообщения от пользователей (включая /start) пересылаются в соответствующий топик

## 🔐 Безопасность

- **Никогда не публикуйте** файл `.env` в репозиторий
- Убедитесь, что `.env` добавлен в `.gitignore` (обычно уже есть)
- Храните токен бота в безопасном месте
- Регулярно проверяйте логи на подозрительную активность
