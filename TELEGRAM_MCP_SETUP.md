# Инструкция по настройке Telegram MCP Server

## ✅ Что уже сделано

1. ✅ Установлен пакет `@chaindead/telegram-mcp` глобально
2. ✅ Добавлена конфигурация в `/home/ae/.cursor/mcp.json`

## 📋 Что нужно сделать вам

### Шаг 1: Получение API ключей Telegram

1. Перейдите на https://my.telegram.org/auth
2. Войдите используя ваш номер телефона Telegram
3. Перейдите в раздел **API development tools**
4. Заполните форму:
   - **App title**: любое название (например, "My MCP Server")
   - **Short name**: короткое название (например, "mcp")
   - **Platform**: выберите "Desktop"
   - **Description**: описание (можно оставить пустым)
5. После создания приложения вы получите:
   - **api_id** (число)
   - **api_hash** (строка)

⚠️ **Важно**: Сохраните эти данные в безопасном месте!

### Шаг 2: Авторизация в Telegram API

Выполните следующую команду в терминале (замените значения на ваши):

```bash
npx -y @chaindead/telegram-mcp auth \
  --app-id ВАШ_API_ID \
  --api-hash ВАШ_API_HASH \
  --phone +7XXXXXXXXXX
```

**Если у вас включена двухэтапная аутентификация**, добавьте параметр `--password`:

```bash
npx -y @chaindead/telegram-mcp auth \
  --app-id ВАШ_API_ID \
  --api-hash ВАШ_API_HASH \
  --phone +7XXXXXXXXXX \
  --password ВАШ_ПАРОЛЬ_2FA
```

**Пример:**
```bash
npx -y @chaindead/telegram-mcp auth \
  --app-id 12345678 \
  --api-hash abcdef1234567890abcdef1234567890 \
  --phone +79991234567 \
  --password my2fapassword
```

После выполнения команды:
1. Telegram отправит вам код подтверждения
2. Введите код в терминал
3. Готово! Сессия будет сохранена

### Шаг 3: Обновление конфигурации MCP

Откройте файл `/home/ae/.cursor/mcp.json` и замените плейсхолдеры на ваши реальные значения:

```json
"telegram-mcp": {
  "command": "npx",
  "args": ["-y", "@chaindead/telegram-mcp"],
  "env": {
    "TG_APP_ID": "12345678",
    "TG_API_HASH": "abcdef1234567890abcdef1234567890"
  }
}
```

**Где найти файл:**
- Путь: `/home/ae/.cursor/mcp.json`
- Или через Cursor: Settings → MCP → Edit Config

### Шаг 4: Перезапуск Cursor

После обновления конфигурации:
1. Полностью закройте Cursor (не просто окно, а приложение)
2. Запустите Cursor снова
3. MCP сервер должен автоматически подключиться

## 🎯 Использование

После настройки вы сможете использовать следующие команды в чате с AI:

### Доступные функции:

1. **`tg_me`** - Получить информацию о вашем аккаунте
   - Пример: "Покажи информацию о моем Telegram аккаунте"

2. **`tg_dialogs`** - Список всех диалогов (чатов, каналов, групп)
   - Пример: "Покажи все мои чаты"
   - Пример: "Покажи только непрочитанные сообщения"

3. **`tg_dialog`** - Получить сообщения из конкретного чата
   - Пример: "Покажи последние сообщения из чата [название]"

4. **`tg_read`** - Отметить диалог как прочитанный
   - Пример: "Отметь чат [название] как прочитанный"

5. **`tg_send`** - Отправить сообщение
   - Пример: "Отправь сообщение 'Привет!' в чат [название]"

### Примеры запросов:

- "Проверь непрочитанные сообщения в Telegram"
- "Покажи последние 10 сообщений из чата [название]"
- "Отправь ответ 'Спасибо!' в последний чат с непрочитанными сообщениями"
- "Суммаризируй все непрочитанные сообщения"
- "Найди чат с [имя] и покажи последние сообщения"

## 🔧 Устранение неполадок

### Проблема: MCP сервер не подключается

1. Проверьте, что вы правильно указали `TG_APP_ID` и `TG_API_HASH` в конфигурации
2. Убедитесь, что выполнили авторизацию (Шаг 2)
3. Проверьте логи Cursor (View → Output → MCP)

### Проблема: Ошибка авторизации

1. Убедитесь, что используете правильный номер телефона (с кодом страны, например +7...)
2. Если у вас 2FA, обязательно укажите `--password`
3. Проверьте, что API ключи правильные

### Проблема: Сессия не сохраняется

Если нужно создать новую сессию, добавьте флаг `--new`:

```bash
npx -y @chaindead/telegram-mcp auth \
  --app-id ВАШ_API_ID \
  --api-hash ВАШ_API_HASH \
  --phone +7XXXXXXXXXX \
  --new
```
